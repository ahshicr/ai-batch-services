# Delivery Standards

## General

- Every project starts with a sample batch.
- Final outputs must include a manifest.
- Failed or low-confidence items must be listed separately.
- Customer data is kept isolated by project.

## Product Images

Deliver:

- Generated images
- Source SKU mapping
- QA notes
- Failed SKU list

Acceptance checks:

- Correct SKU mapping
- Correct dimensions
- No visible watermark
- No obvious product deformation
- Style matches approved sample

## Offline Data Processing

Deliver:

- Structured output file
- Failed-row file
- Sample audit file
- Schema notes

Acceptance checks:

- Valid CSV/JSON/Excel structure
- Required fields present
- Sample audit meets agreed accuracy target

## RAG Knowledge Base

Deliver:

- Cleaned text chunks
- Embedding/vector store
- Test questions
- QA report
- Setup notes

Acceptance checks:

- Source references are preserved
- Answers stay within document scope
- Retrieval examples are inspectable
